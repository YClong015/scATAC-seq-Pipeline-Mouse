# Stage 04 - Universal peak set

Combines the four per-tissue consensus peak sets (from stage 03) into a single cross-tissue universal peak set spanning all four tissues. This is the substrate for all downstream cross-tissue analysis (DAR, HOMER, aging comparison).

## Method

`merge_universal_mm10.py` uses pycisTopic's `get_consensus_peaks()` to iteratively merge the four per-tissue consensus BEDs:

1. Read each tissue's `consensus_regions.bed.gz` from stage 03.
2. Treat each tissue's consensus BED as a "sample" input to `get_consensus_peaks()`.
3. Iterative greedy merging with 250-bp half-width, mm10 blacklist removal.
4. Write `consensus_regions.bed.gz` to a unified directory.

## Run

```bash
(cd 04_universal_peaks && sbatch merge_universal_peaks.slurm)
```

The SLURM wrapper sets environment variables for the python script:
- `PEAK_HALF_WIDTH=250`
- `BLACKLIST=/scratch/user/$USER/.../mm10-blacklist.v2.bed`
- `CHROMSIZES_FILE=/scratch/user/$USER/.../mm10.chrom.sizes`
- `PEAKPATH=${UROOT}/tissue_consensus_paths.txt` - the manifest of 4 tissue BEDs
- `OUTDIR=${DATA_ROOT}/Universal_peaks/Kidney_Aorta_Tcells_Lung/consensus_peak_calling`

## Outputs

| File | Contents |
|---|---|
| `${DATA_ROOT}/Universal_peaks/Kidney_Aorta_Tcells_Lung/consensus_peak_calling/consensus_regions.bed.gz` | **The canonical universal peak set used for DAR.** All stage-05 per-tissue re-quantifications read this BED. |
